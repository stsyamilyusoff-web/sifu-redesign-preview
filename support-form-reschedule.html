Support · Reschedule a class (form)

The class-reschedule ticket, for a class too old or already attended to move on the calendar. Canonical form pattern (centered nav, grouped rows, picker sheets, sticky dock). It restructures the live 6-field flow into 3 tidy groups (which class · new time · reason) without dropping a field. Naming collision handled: the field links a job request (stays "request", the sheet lists your requests) but the form creates a support ticket (the button is "Submit ticket"). The new end time is auto-set from the class duration (read-only). Submitting opens the Parent approval confirmation, it is not an instant send. Dates one format DD-MMM-YYYY. No duplication, no em-dashes. True 402×874, light + dark.
Tint = submit + selected option + back Neutral = fields, values, the auto end time
