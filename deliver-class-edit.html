Deliver · Class Edit — extend · postpone · parent-approval gate (2026-07-01)

Three screens on the class-management J2 journey. All three live downstream of the Class Records detail or the Active Class cockpit.
Design decisions per audit brief:
· ExtendClass — quick-pick chips are the focal control (30–120 min, 8 options in 2 rows); one-time-only notice in amber; result preview card updates with the new end time + duration. One tap = done.
· EditPostponedClass — FROM → TO comparison card leads the screen so the tutor sees exactly what is changing before editing anything. Reason field is a multiline text area (real sample text). Max +30 days note under the date row.
· ParentApprovalConfirmation — a neutral friction gate (not a success screen, not a dark pattern). Two equally-prominent buttons: the confirm CTA and a full-width secondary "Wait, let me check" so declining is just as easy as confirming.
Colour budget (locked): primary CTA = brand-blue (--tint), never green. Green = positive status only. Amber = one "check this" flag. Neutral copy, no em-dashes, no invented backend values. Date format DD-MMM-YYYY. Money whole-RM on context cards. Icons are emoji placeholders. No raw hex in frame markup.
