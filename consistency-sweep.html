Consistency sweep
Every remaining screen (~40-50) brought to the design floor. The gaps are systemic, so this is 7 rules applied app-wide, not 50 bespoke fixes. A build spec, with reference screens to copy. For Hafiz.
Approach: a token + copy migration of the older screens to the v2 system. No layout redesign, no new screens. All gaps are cosmetic + copy (no functional bugs found in the audit). Three screens already do it right; the build copies their pattern.
The 7 rules (apply app-wide)
1
Fonts to the constant
Replace every hard-coded 'Circular Std Book/Medium/Bold/Black' with Fonts.book/medium/bold/black. The biggest gap, in nearly every older screen.
2
Hex to theme tokens
#E6F2FF to PattensBlue, #003E9C to SmaltBlue, #298CFF33 to darkGrayLite, #ececec to WhiteSmoke, #CECECE to shinyGrey, Google blue #1a73e8 to brand #243CB3, Theme.red to Theme.Red.
3
Card radius to 12
Standardize the jumble of 5 / 8 / 10 / 15 / 16 / 18 / 20 / 30 to the system 12px.
4
Copy
Fix typos ("Avaiable" to "available", "Exprience" to "Experience"); remove emojis from strings; sentence case (not "KNOWLEDGE" or Title Case); move hard-coded titles / buttons / placeholders into translations.
5
One date format
Use DD MMM YYYY via dayjs everywhere; fix the MMM DD / toLocaleDateString / date-fns mismatches.
6
Remove debug logs
Drop the leftover logger.log / console.log (EvaluationReportView, ProgressReportView, ReferFriend, NewsDetails).
7
Rich empty states
Icon + line, not bare text. Use the IntroVideoUpload empty state as the model.
