Deliver · Success/confirmation screen variants (2026-07-01)

These are the 5 success variants brought to one quality floor. Founder direction: variety is intentional — each confirmation ends a different journey so the screen should feel distinct, not copy-pasted. Do not collapse them to a single template.

Convention (locked this session):
  • Primary CTA = brand-blue (--tint). Never green. Green is reserved for the success check icon only (--green-weak fill, --green-ink text).
  • Neutral copy — no hype, no em-dashes, no double-exclamation. "Sifututor" if the brand name appears.
  • All visual values are CSS-variable tokens — no raw hex in the frame markup.
  • Variants 3 and 4 include dark-mode frames (the metadata + secondary-action variants — the most complex, so most worth validating in dark).

All sample data is labelled. Icons are emoji stand-ins for SF Symbols in the real build.

Frame headroom notes (content height vs 874px device frame):
V1 ClassPostponeSuccess — ~58px headroom (compact, no extra rows).
V2 ClassScheduleAdded — ~90px headroom (fewest elements, most breathing room).
V3 BackdateEscalationSuccess — ~34px headroom (notice chip + reference chip make it the tallest).
V4 EvaluationReportSuccess — ~18px headroom (tightest; secondary .subrow button + hint + sumcard).
V5 ProgressReportSuccess — ~70px headroom (sumcard with 2 rows, no secondary action).
