Activate · Set your location (quick-pin picker)

The BioDetails location step, the primary path (founder-approved 2026-07-01): instead of typing, the tutor drops a map pin (the Grab / foodpanda pattern). GPS pins their current spot, they nudge the map to fine-tune, and the pin reverse-geocodes into address + state + city + postal, the same fields (and the same lat/long the live app already stores). Type-to-search stays as the fallback. This is the tutor's home address, captured for verification (where they teach is set separately in "Where & how you teach", by city). Never shown to parents.
Folded into BioDetails as the primary location path (see activate-identity-forms.html). This picker is its own screen · a new node. Needs a native map SDK (Google Maps / Mapbox) · a build + Hafiz item, not a JS-only change.

Exploration notes

1
Same data, less friction. The pin produces the exact lat/long the live app already stores (Google Places gives it today), plus a reverse-geocoded address + state + city + postal, so nothing downstream changes. It just replaces typing with a tap-and-nudge.
2
Type stays as the fallback. Some tutors prefer typing, or GPS is off, or the pin is wrong. The "Type my address instead" path keeps the current autocomplete + manual fields, so nobody is blocked.
3
Auto-filled fields stay editable. State / City / Postal show an "Auto" tag but remain drill-in editable, in case the reverse-geocode is slightly off. Never a locked black box.
?

Privacy: the exact home pin is sensitive · keep it verification-only, never parent-visible. Request matching stays city-based in Step2 (Where & how you teach), not this pin.
