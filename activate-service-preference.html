Field-level audit + fidelity fixes (verified vs live code)

1
Curricula is now multi-select. Live (Step1WhatYouTeach) reads data.curricula[] as a multi-select deduped by id; the prior mock showed a single "Choose curriculum". Fixed to removable chips.
2
Per-language proficiency. Live stores a proficiency for EACH selected language (Native / Fluent / Conversational, defaults conversational). The prior mock showed one shared row; now each language has its own.
3
Mode + availability corrected. Mode = Online / In-person / Hybrid (live enum; was "Both"). Step 3 "Open to other times too" is a toggle with the live subtitle (was a 6th slot), and the 5 presets keep their live descriptions ("The dominant Sifututor slot", etc.).
5
Cities → teaching area (founder direction 2026-07-01). The live city multi-select is replaced by a pin + travel-radius service area (distance-based matching), gated on mode: Online = no location (fixes the live "cities required for Online" oddness), In-person / Hybrid = the area. Multiple areas allowed. The "Set your teaching area" picker is its own node (activate-teaching-area.html).

(d) proficiency defaults to "conversational" if untouched.
